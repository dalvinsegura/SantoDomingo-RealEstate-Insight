¿Por qué este proyecto?

🗂️
Mercado fragmentado

Datos dispersos en múltiples portales (clasificados.com.do, supercasas.com, etc.)

📊
Sin comparación de precios por sector

No existe una herramienta accesible que muestre qué sectores son más caros o baratos

❓
¿Este precio es justo?

Imposible saberlo sin años de experiencia en el mercado local

⏱️
Tiempo perdido

Compradores y corredores navegan decenas de páginas para recopilar información básica

💡 Nuestra propuesta: centralizar, normalizar y analizar esa información en un solo lugar.

¿Qué construimos?

🕷️

Scraping

Extrae propiedades de portales reales automáticamente y las normaliza

⚙️

Backend API

Almacena, valida y expone los datos mediante 8 endpoints REST

🖥️

Dashboard

Gestiona, busca, calcula precios y visualiza el mercado en un mapa interactivo

Tres capas independientes que trabajan juntas — el usuario final solo ve el dashboard.

Problemas que resolvimos

🟡 Leaflet + SSR Next.js → dynamic(import, {ssr:false}) con dos componentes separados
🟡 Columna estado faltante → migrate_scraping_columns() corre automáticamente al importar el módulo
🟡 Estructura respuesta /api/stats → Ajuste del envelope: { ok, stats: { total, precio_promedio, ... } }
🟡 Closure stale en calculadora → setResultado(null) al inicio de cada cálculo + eliminación del useCallback
🟡 Sectores con nombres inconsistentes → Matcher fuzzy en 3 niveles: exacto → substring → token match (≥4 chars)
